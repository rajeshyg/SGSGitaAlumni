# Session Logs Directory

This directory stores session analysis JSON files generated by the `stop-session-analyzer.js` hook.

## File Naming Convention

Files are named: `{timestamp}_{session_id}.json`

Example: `2025-12-02T10-30-00-000Z_abc123.json`

## Analysis Contents

Each session log contains:

- **files_read**: Files read during the session
- **files_modified**: Files edited during the session  
- **files_created**: New files created
- **commands_run**: Terminal commands executed
- **searches_performed**: Search queries (grep, glob, semantic)
- **tool_calls**: Detailed log of all tool invocations
- **tool_summary**: Count of each tool type used
- **framework_violations**: Any rule violations detected
- **framework_checks**: Status of each compliance check
- **stats**: Session statistics (duration, tool counts)

## Viewing Session Logs

1. **CLI (PowerShell)**:
   ```powershell
   # View latest session
   Get-Content (Get-ChildItem .claude/session-logs/*.json | Sort-Object LastWriteTime -Descending | Select-Object -First 1)
   
   # Find sessions with violations
   Get-ChildItem .claude/session-logs/*.json | ForEach-Object { 
     $json = Get-Content $_ | ConvertFrom-Json
     if ($json.framework_violations.Count -gt 0) { $_.Name }
   }
   ```

2. **Visual Dashboard**:
   Open `.claude/session-viewer.html` in a browser for interactive viewing.

## Retention

Session logs are not auto-deleted. Periodically clean old logs:

```powershell
# Delete logs older than 30 days
Get-ChildItem .claude/session-logs/*.json | Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) } | Remove-Item
```
